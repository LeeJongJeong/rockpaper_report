# 락페이퍼 주간업무보고 분석 대시보드

업무일지 형태의 엑셀 파일을 드롭/업로드하면, 필터 조합과 탭 전환으로
지표·트렌드·인사이트를 빠르게 조회할 수 있는 정적 대시보드입니다.

## 현재 기준 요약

- 빌드 과정 없음 (순수 정적 웹 앱)
- 파일 직접 업로드 지원 (로컬 `.xlsx`, `.xls`, 다중 파일 병합)
- 기간 필터 + 다중 항목 필터(부서/엔지니어/제품/지원유형/고객사/담당영업)
- 차트 드릴다운 + 테이블 검색/정렬/다운로드
- 자동 인사이트 카드(최대 15개, 룰 기반 경보/권고)
- 소정근무시간 자동 계산(영업일 × 8h, 주말·공휴일 제외)
- 근무-보상시간 통계 시트 기반 “총 보상발생시간 Top3 집중도” 인사이트 반영

## 빠른 시작

1. 브라우저로 `index.html` 열기
2. 업로드 영역에 주간업무보고 엑셀 파일 업로드
3. 기간/필터/탭을 바꿔서 KPI, 차트, 자동 인사이트 확인


## 폴더 구조

- `index.html` — 레이아웃, CDN 스크립트, 탭/필터 UI, 스크립트 로더
- `src/config.js` — 공통 설정값/색상/컬럼/제품군 규칙/공휴일 규칙
- `src/utils.js` — 날짜 파싱, 포맷, 숫자 포맷, 토스트, 로딩 오버레이, 디바운스
- `src/contract-utils.js` — 소정근무일/시간 계산 로직(휴일 반영)
- `src/contract-utils.test.js` — 소정근무 계산 회귀 테스트
- `src/app.js` — 전체 애플리케이션 로직(파싱·집계·차트·필터·인사이트)
- `src/styles/main.css` — 전체 스타일
- `주간업무보고_*.xlsx` — 샘플 데이터 파일

## 핵심 기능

### 1) 업로드/병합

- `.xlsx`, `.xls` 파일 1개 이상 업로드
- 기존 데이터 대체 또는 추가 업로드 지원
- 헤더는 템플릿 기준으로 `sheet_to_json(..., { range: 2 })` 사용(4행부터 데이터 반영)

### 2) 기간 및 다중 필터

- 기간: Flatpickr 기반 `from ~ to` 날짜 범위
- 다중 필터: 부서/엔지니어/제품/지원유형/고객사/담당영업
- 필터 변경 시 실시간 재집계(디바운스 150ms)

### 3) 탭별 분석

- Overview
  - 전체 KPI, 일별 지원 추이(7일 이동평균), 주차별 추이, 제품/지원유형/부서 분포, 요일분포, 팀별 가동률
- 엔지니어
  - 엔지니어별 가동률, 투입량, 고객사/제품 커버리지, 과부하/의존도 신호 등
- 제품
  - 제품별 지원 건수·효율, 제품군 집계, 제품별 고객/영향도 분석
- 지원유형
- 고객사
- 담당영업
- 상세 테이블
  - 테이블 검색, 정렬, 페이지네이션, 필터 결과 엑셀 내보내기

### 4) 차트 드릴다운

- 파이/막대/히트맵 일부 차트는 클릭으로 특정 값에 대한 필터를 단건 적용
- 적용된 드릴다운은 배너로 표시, 개별 해제 가능

### 5) 자동 인사이트

- 생성 조건 충족 시 최대 15개 카드 표시
- 대표 항목 예시
  - 총 보상발생시간 Top3 집중도
  - 전체 가동률 경보/달성
  - 개인 가동률 경보(낮은 가동률 + 팀평균 대비 이격)
  - 업무 집중 리스크, 고객사 단일 의존도, 제품 커버리지 위험
  - 과부하/내부업무 과다/저활동 엔지니어/Presales 활성도 등
- `renderInsightPanel`에서 카드 타입(`danger/warning/success/info`)으로 렌더링

## 소정근무 계산(핵심)

- 기준: `소정근무시간 = 영업일 수 × WORK_HOURS_PER_DAY(기본 8h)`
- 영업일 수는 다음을 제외:
  - 주말(토/일)
  - 휴일(`HOLIDAYS`에 정의된 fixed/yearly/extras)
- 휴일의 대체공휴일(월~금 중 다음 평일)도 옵션으로 반영 가능
- 월 단위 구간 요약도 제공(필터 기간에서 월별 상세 계산)

설정 위치: `src/config.js`의 `CONFIG.HOLIDAYS`, `CONFIG.WORK_HOURS_PER_DAY`

## 데이터 입력 요구사항

### 메인 시트(필수)
- 4번째 행(인덱스 3)부터 데이터(현재 코드 기준)
- 주요 컬럼명:
  - `작업시작일시`, `작업종료일시`, `부서명`, `엔지니어`, `제품명`, `지원유형`, `고객사명`, `담당영업`  
  - 계산/표시용: `작업시간(h)`은 런타임에서 계산되어 입력

### 근무-보상시간 통계 시트(옵션)
- 시트명 예시: `근무-보상시간 통계` / `근무-보상시간` / `보상` 포함명
- 사용 컬럼 예시:
  - `엔지니어`, `팀명`, `총 보상발생시간`, `근무시간`, `대체 근무시간`, `총 근무시간`
  - 최초 행에 `YYYY-MM-DD ~ YYYY-MM-DD` 형태 기간 정보가 있으면 필터 기간 교집합 시 계산

## 테스트

### 회귀 테스트(회사가수 휴일/월별 소정근무)

```bash
node src/contract-utils.test.js
```

- 예시 검증 항목
  - 2026-01-01 ~ 2026-02-28 영업일 38일, 소정근무 304h
  - 월별 분해(`byMonth`) 존재/키 정합성
  - 정규화 전/후 설정 동일성 비교

## 커스터마이징 포인트

- 임계치/차트 탑N 변경: `src/config.js`
- 지원유형 분류(가동률 포함 여부/카테고리): `src/app.js`
- 제품군 매핑 규칙: `src/config.js`의 `PRODUCT_GROUP_RULES`
- 공휴일 규칙 확장: `src/config.js`의 `CONFIG.HOLIDAYS`

## 최근 변경 포인트(운영 상 주의)

- 주별 집계 기준은 월요일 시작 주차를 기준으로 버킷화(`getWeekKey`)
- 자동 인사이트는 최우선으로 경보성이 높은 항목부터 채움(최대 15개)
- 필터/드릴다운 모두 `rawData` 재필터링 기반으로 동작하므로, 파일 추가/리셋 시 상태 초기화 필요

## 운영 가이드

- 권장: 월말/분기말 데이터 점검 시 공휴일 규칙(`HOLIDAYS`)을 먼저 갱신
- 데이터 컬럼명이 기존 명칭과 다를 경우 자동 인식이 실패할 수 있음(특히 보상 시트)
- 대용량 파일 업로드 직후에는 브라우저 처리 시간으로 인해 로딩 오버레이가 잠시 표시됨

## 라이선스/보안

- 본 저장소는 조직 내부 운영용 정적 분석 보드로, 민감 데이터는 로컬 브라우저에서만 처리됨
- 업로드 파일은 원격 업로드하지 않으며, 브라우저 메모리 내에서만 가공됨
